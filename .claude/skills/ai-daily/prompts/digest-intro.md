# 日报组装 Prompt

你正在将各板块的摘要组装成一份完整的 AI 日报。

## 格式

以此标题开头（替换日期）：

```
# AI 日报 · YYYY-MM-DD

> 采集时间 HH:MM | 信息源: X | 条目: X
```

## 板块顺序

1. **重大新闻** — 当天最重要的 3-5 条，跨所有来源选出
2. **公司动态** — AI 公司官方博客和媒体报道
3. **Twitter/X 精选** — 来自监控清单的重要推文
4. **论文与开源** — arXiv、GitHub Trending、HuggingFace
5. **社区热点** — HackerNews、Reddit、V2EX 等

每个板块使用 `## 板块名` 作为标题。

## 每条内容的格式（严格遵守）

每条内容 **必须** 包含四个要素：

```
### 标题

**来源**: 来源名称
**链接**: [原文链接](url)

事件描述，100-200字。
```

## 强制规则

- **没有链接的内容不收录** — 没有 URL 就不是真实内容，不要包含
- **不要编造** — 所有信息必须来自采集数据的 JSON，不要猜测、推测、补充
- **不要模糊概括** — 每条摘要要有具体的数字、人名、产品名
- **合并重复** — 同一事件被多个源报道时，合并为一条，选信息最丰富的源
- 如果某个板块没有内容，**跳过该板块**，不要写"暂无"
- Twitter 板块如果采集失败，注明"Twitter 数据未采集（opencli 未连接）"

## 结尾

在最后添加采集统计表格：

```
---

## 采集统计

| 指标 | 数值 |
|------|------|
| 信息源总数 | X |
| 成功采集 | X |
| 条目总数 | X |
| 去重移除 | X |
```
